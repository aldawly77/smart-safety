Embedded Engineering & Circuit Wiring

ESP32 / Arduino Microcontroller Schematics

Pin Mappings & Communication Protocol:

  • Green LED (Pin 11): Stays ON when the camera detects complete safety compliance (Helmet + Vest).
  • Yellow LED (Pin 10): Flashes periodically to warn of minor safety breaches (e.g., Vest only, missing Helmet).
  • Red LED (Pin 9) & Buzzer (Pin 8): Continuous siren alarm and rapid red strobe blinking for critical safety hazards.
  • Serial UART Stream: C++ sends single character status flags ('S'=Safe, 'W'=Warning, 'D'=Danger) to the board over USB.

Microcontroller Firmware (Embedded C++):

void loop() { if (Serial.available() > 0) { char state = Serial.read(); if (state == 'S') digitalWrite(GREEN_LED, HIGH); // Safe compliance if (state == 'D') tone(BUZZER_PIN, 1100); // Trigger buzzer siren } }

Graduation Panel Anticipated Q&A Guide

Academic Defense Strategies

Q1: Why did you opt for a hybrid C++ (Qt) and Python computer vision architecture?

Engineering Defense: Python offers rapid prototyping and extensive libraries for artificial intelligence and machine learning (such as OpenCV and YOLO neural models). However, C++ (Qt) is heavily optimized for desktop graphics, event handling loops, low-level serial communication protocols, and performance efficiency. Using QProcess/IPC combines both worlds: robust, real-time computer vision coupled with a lightning-fast, production-grade desktop controller.

Q2: How does your class design prevent memory leaks, especially with polymorphic pointers?

Engineering Defense: All polymorphic base classes (such as `Device` and `Detector`) define **virtual destructors** (`virtual ~Device()`). This ensures that when a derived object is deleted via a base pointer, the compiler dynamically binds and executes the derived destructor first, followed by the base destructor, reclaiming all dynamically allocated resources cleanly.

Q3: How are safety helmet and vest colors filtered stably?

Engineering Defense: The frame images captured by the CameraSystem are translated from the default RGB colorspace to **HSV (Hue, Saturation, Value)** colorspace. Unlike RGB, HSV decouples brightness (V) from color hue (H), enabling highly robust threshold masking of yellow helmet regions and green vest fabrics under volatile ambient lighting conditions.
